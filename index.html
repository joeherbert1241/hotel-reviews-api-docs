Hotel Reviews API

Welcome to the Hotel Reviews API documentation. Find all the information you need to integrate with our services.

Base URL: https://hoteladvisor.sales-engineering.matillion.com

Authentication

The Hotel Reviews API uses API Keys to authenticate requests. You must include an API key in the header of your requests.

Header Name: x-api-key

Header Value: qU7FL6LxAT2VVkT9F8oTH4j63r9uDDXa6FESKA6w (Your API Key)

Important: Keep your API key secure and do not share it publicly. All API requests must be made over HTTPS.

GET

/hotel-reviews/{reviewId}

Retrieves a specific hotel review by its ID.

Path Parameters:

  • reviewId (string, required): The unique identifier for the hotel review.

Request Headers:

x-api-key: YOUR_API_KEY

Example Request (cURL):

curl -X GET "https://hoteladvisor.sales-engineering.matillion.com/hotel-reviews/review_123" \
 -H "x-api-key: qU7FL6LxAT2VVkT9F8oTH4j63r9uDDXa6FESKA6w"

Example Success Response (200 OK):

{
  "id": "review_123",
  "hotelId": "hotel_abc",
  "rating": 5,
  "comment": "Excellent stay, great service!",
  "author": "John Doe",
  "createdAt": "2023-10-26T10:00:00Z",
  "updatedAt": "2023-10-26T10:00:00Z"
}

Example Error Response (404 Not Found):

{
  "error": {
    "code": "NOT_FOUND",
    "message": "Hotel review with ID 'review_xyz' not found."
  }
}
POST

/hotel-reviews

Creates a new hotel review.

Request Headers:

x-api-key: YOUR_API_KEY
Content-Type: application/json

Request Body (JSON):

{
  "hotelId": "hotel_def",
  "rating": 4,
  "comment": "Very comfortable rooms and friendly staff.",
  "author": "Alice Brown"
}

Fields like rating (e.g., 1-5) and comment are typically required. Specify actual requirements.

Example Request (cURL):

curl -X POST "https://hoteladvisor.sales-engineering.matillion.com/hotel-reviews" \
 -H "x-api-key: qU7FL6LxAT2VVkT9F8oTH4j63r9uDDXa6FESKA6w" \
 -H "Content-Type: application/json" \
 -d '{
   "hotelId": "hotel_def",
   "rating": 4,
   "comment": "Very comfortable rooms and friendly staff.",
   "author": "Alice Brown"
 }'

PUT

/hotel-reviews/{reviewId}

Updates an existing hotel review by its ID.

Path Parameters:

  • reviewId (string, required): The unique identifier for the hotel review to update.

Request Headers:

x-api-key: YOUR_API_KEY
Content-Type: application/json

Request Body (JSON):

{
  "rating": 5,
  "comment": "Updated: Absolutely fantastic experience!"
}

Include only the fields you want to update. Behavior for partial updates (PATCH vs PUT) should be clarified.

Example Request (cURL):

 -H "x-api-key: qU7FL6LxAT2VVkT9F8oTH4j63r9uDDXa6FESKA6w" \
